    • Conceptos Básicos de Git

    Git init

    Este comando inicializa un nuevo repositorio Git en un directorio existente o vacío. Crea una nueva subcarpeta llamada .git que contiene todos los archivos necesarios del repositorio.


    Git clone

    Este comando clona un repositorio Git existente desde una ubicación remota (como GitHub) y lo copia en tu máquina local en un nuevo directorio.


    Git add .

    Este comando añade todos los archivos modificados y nuevos al área de preparación (staging area) para ser incluidos en el próximo commit.


    Git commit -m

    Este comando guarda los cambios realizados en el área de preparación en el repositorio. El parámetro -m se utiliza para añadir un mensaje al commit que describe los cambios realizados.


    Git status

    Este comando muestra el estado actual del repositorio, incluyendo los cambios realizados, los archivos en el área de preparación y los archivos que aún no han sido seguidos por Git.


    Git push

    Este comando sube los cambios locales confirmados a un repositorio remoto, como GitHub. Se utiliza para actualizar el repositorio remoto con los cambios locales.


    Git pull

    Este comando descarga los cambios desde un repositorio remoto y los fusiona con el repositorio local. Es útil para sincronizar tu repositorio local con los cambios realizados por otros colaboradores en el repositorio remoto.


    Git branch

    Este comando muestra la lista de ramas en el repositorio actual, así como permite crear, eliminar y cambiar entre ramas.
